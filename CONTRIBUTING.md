## Contributing

Thanks for your interest in contributing to Sales Funnel Analysis!

We welcome contributions of all kinds: bug reports, feature suggestions, documentation improvements, tests, and code. To make collaboration smooth and efficient, please follow these steps:

1. Fork the repository
2. Create a new branch for your changes: `git checkout -b fix/brief-description`
3. Make small, focused commits (one logical change per commit)
4. Add tests for new functionality or behavior, when applicable
5. Run tests locally: `pytest -q`
6. Push your branch and open a pull request describing the change and motivation

### Code style and tests
- Use consistent formatting (e.g., black or `python -m pip install black`)
- Add unit tests where possible
- Keep changes small and focused

### PR review process
- PRs are reviewed and commented on by project maintainers. Please respond to review comments and make updates.
